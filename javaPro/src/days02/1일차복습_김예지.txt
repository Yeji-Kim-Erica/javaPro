1. JDK/JRE/JVM에 대해서 설명하세요.

JDK는 Java Development Kit의 약자로, 자바 개발을 용이하게 해주는 프로그램이며, 오라클에서 제공하고 있습니다.
JDK = JRE + 개발도구 (bin 폴더의 컴파일러, 실행기...예: java.exe, javac.exe)
JRE는 Java Runtime Environment의 약자로, 자바 개발 환경을 말합니다.
JRE = JVM + 클래스 라이브러리
JVM은 Java Virtual Machine의 약자로, javac에서 컴파일한 소스파일, 즉, 바이트코드 파일을 다시 기계어로 번역해 java 명령어를 통한 프로그램의 실행을 담당하고 있습니다.

2. JDK 설치 확인 방법에 대해서 설명하세요.

- 검색창에 cmd를 입력해 명령 프롬프트를 띄웁니다.
- 'javac -version'을 입력합니다.
- 설치한 파일의 버전이 일치하는지 확인합니다.
- 오류 메시지가 뜰 경우, 환경 변수 설정에 문제가 있으므로 다시 수정합니다.

3. 설치된 JDK 제거하는 방법에 대해서 설명하세요.
: 프로그램 제거를 검색. 들어가서 java 검색. 제거.

4. JDK 를 설치 후 환경변수 설정에 대해서 설명하세요.  
  [시스템변수] 
  1) 변수이름 	: [ JAVA_HOME ]
      변수값    	: [ C:\Program Files\Java\jdk-11 ]

5. 자바에서 
  1) 클래스 선언 형식을 적으세요.
[접근지정자] [기타제어자] class 클래스명 { 클래스 몸체 }
   
  2) 함수(메서드) 선언 형식을 적으세요.
[접근지정자] [기타제어자] 리턴자료형 메서드명 ( [매개변수...] ) {

자료형 변수명 [= 초기값] ;

6. 자바 프로그램의 기본 구조에 대해서 설명하세요.
: OOP(객체 지향). 자바는 클래스를 기본 단위로 가지고 있고, 메서드를 이용해 실행합니다.

- package 선언(선택): 패키지명은 소문자로 작성되어야 함.
- import (선택)
- 시작 개체(필수): 반드시 하나의 main 메서드를 가진 클래스가 선언되어야 함.
- main 메서드(필수)

9. 키워드(keyword)와 식별자( identifier) 설명하세요.
 - 키워드: 이미 자바에 저장되어 정해진 역할이 있는 예약어로서, 식별자로 사용 될 수 없다. (예: class, public...)
 - 식별자: 개발자가 직접 명명하여 의미를 부여하는 이름을 말한다. (예: 변수명, 클래스명, 패키지명...)
 
10.  [변수, 상수, 리터럴]에 대해서 설명하세요. 
   - 변수: 값을 여러번 할당할 수 있는 기억공간
   - 상수: 값을 1번만 할당할 수 있는 기억공간
   - 리터럴: 고정된 값 (예: 'A', 9, true, 3.14)

11. 식별자(변수,클래스명, 패키지명 등등)를 명명하는 규칙에 대해서 설명하세요. 
 - 알파벳, 숫자, 밑줄, 달러 기호로만 구성된다.
 - 맨 첫 글자에 숫자가 올 수 없다.
 - Camel Case 규칙에 따라 작성하되, 클래스가 아닌 경우 첫 글자에는 대문자가 올 수 없다. 클래스의 경우 첫 글자는 대문자로 작성한다.
 - 대소문자가 구분된다.

12. 함수(메서드)를 사용할 때 중요한 3가지 사항에 대해서 설명하세요.
     1)  기능: 어떤 일을 하는지
     2)   매개변수(파라미터): 일(기능)을 하기 위해 필요한 값
     3)   리턴값(리턴자료형): 일이 끝난 후 돌려줄 값 (main메서드의 경우, 리턴값이 void이기 때문에 코드에서 생략됨)
