1. 필드(멤버변수)를 초기화 하는 방법 및 우선 처리 순위에 대해서 설명하세요
	
	1. 명시적 초기화: 필드에서 바로 선언
	2. 초기화 블럭: 블럭 안에 선언
		1) static 초기화 블럭 : 클래스 변수들을 일괄적으로 초기화하기 위한 블럭. 1번만 실행됨.
		2) 인스턴스 초기화 블럭 : 생성자에서 중복으로 적힌 코딩을 한번만 작성하기 위한 블럭. 객체 생성시 마다 실행됨.
	3. 생성자 초기화: 생성자를 이용헤 초기화
	
	- 클래스 변수: 기본값 -> 명시적 초기화 -> static 초기화 블럭
	- 인스턴스 변수: 기본값 -> 명시적 초기화 -> 인스턴스 초기화 블럭 -> 생성자

2. 클래스 간의 관계에 대해서 설명하세요 .
 1) has-a 관계 (소속관계) : 한 클래스가 다른 클래스의 부품으로 속해있는 관계
 2) is-a 관계 (상속관계) : 부모클래스의 코드를 재사용해서 새로운 자식 클래스를 선언하는 것
 
3. 상속(inheritance)에 대해서 설명하세요 
  ㄱ. 정의 :  기존의 클래스를 재사용해서 새로운 클래스를 선언하는 것
  ㄴ. 장점 :  코드의 재사용으로 중복코딩을 할 필요가 없어짐, 생산성이 향상되고 유지 및 보수가 용이함
  ㄷ. 선언형식 : class 클래스명 extends 부모클래스명 { }
   
4. 모든 클래스의 최상위 부모클래스는 ?
	: Object
	(java.lang.Object)

5. 오버로딩(overload)과 오버라이딩(override)에 대해서 설명하세요.
	- 오버로딩(Overload) : 중복 함수. 동일한 메소드명을 가지고 있으나, 매개변수를 달리하여 함수를 여러번 선언하는 것을 말함.
	- 오버라이딩(Override) : 재정의 함수. 동일한 메소드명, 매개변수, 리턴자료형을 가지나 그 실행 내용을 새롭게 재정의한 것을 말함.

6. this와 super 키워드에 대해서 설명하세요 . 
	- this : 해당 객체의 주소값을 값으로 가지는 참조변수
	- super : 해당객체의 부모멤버를 가리키는 참조변수

7. 단일 상속과 다중 상속에 대해서 설명하세요 .
	- 단일 상속은 한 클래스당 하나의 클래스만 상속받을 수 있는 것을 말함.
	- 다중 상속으로 한 클래스 당 여러개의 클래스에서 상속을 받을 수 있게 되면,
	해당 클래스 내에서 부모멤버를 호출할 시 멤버명이 동일한 경우에는
	어느 클래스의 부모멤버를 가리키는 건지 알 수 없기 때문에 자바에서는 단일 상속만 가능.
	인터페이스를 사용해 대체할 수 있다.

8. @Override 애노테이션(Annotation)에 대해서 설명하세요.
	- 어노테이션: 코드에 메타데이터를 추가하는 방법
		ㄴ 메타데이터: 정보를 담는 것
	상속관계에서 오버라이딩을 할 때 오버라이딩을 하기 위한 조건에 부합하게 작성되었는지,
	오버라이딩이 잘 되었는지를 확인할 수 있게 해주는 어노테이션.
	
	- 프로그램 실행이 너무 잘 안될 때 metadata 폴더를 지우고 이클립스를 실행해보자..
	
	자바의 어노테이션은 코드에 메타데이터를 추가하는 방법입니다.
	어노테이션은 클래스, 메소드, 변수, 파라미터, 패키지에 추가할 수 있으며 주로 컴파일러나 런타임 실행시 특정 동작을 유도

9. [오버라이딩 조건]에 대해서 설명하세요 .
	1. 메소드명이 동일
	2. 매개변수가 동일
	3. 리턴자료형이 동일
	4. 부모클래스와 범위가 같거나 더 넓은 접근지정자만 사용 가능
	5. 부모클래스와 같거나 더 적은 예외처리만 가능
	6. 인스턴스 멤버는 static 멤버로 바꿀 수 없고,
		static 멤버는 인스턴스 멤버로 바꿀 수없음
	7. final 키워드가 붙은 메소드는 오버라이딩 불가

10. final 키워드에 대해서 설명하세요.
	- final class 클래스 선언
	- final 메서드 앞: 오버라이딩을 할 수 없는 최종 메서드
	- final 필드: 
	- final 지역변수
	더 이상 내용을 변경할 수 없도록 하기 위해 붙이는 키워드.
	변수에 final을 붙이면 상수가 되어 그 기억공간의 값이 고정되어 변경할 수 없게 되고,
	메소드에 final을 붙이면 해당 클래스의 자식 클래스에서 해당 메소드의 실행 내용을 변경할 수 없게 됨
	
	- final 필드 예시
	class 클래스명, text

11. 클래스들 간의 형변환 (클래스들간의 형변환은 상속 관계에 한해서만 가능하다.)
	업캐스팅( upcastring)과 다운 캐스팅( downcastring )에 대해서 설명하세요.
	부모 클래스와 자식 클래스 사이에서 일어나는 변환 작업.
	자동 형변환과 비슷한 개념.
	자식클래스에서 객체를 새로 생성해서 부모클래스에 참조시키는 것을 업캐스팅(upcasting)이라고 함.
